1. OVERVIEW
================================================================================

This document outlines the scope of work for building a PII (Personally 
Identifiable Information) redaction tool that can detect and redact sensitive 
information from both plain text and PDF files. The tool will demonstrate 
responsible AI practices while providing a user-friendly interface for 
protecting sensitive data.

================================================================================
2. WHAT WILL BE IN SCOPE
================================================================================

2.1 Core Functionality
----------------------
✓ Accept text input (via text area or file upload)
✓ Accept PDF file uploads
✓ Detect PII types:
  - Email addresses
  - Phone numbers (various formats: US, international)
  - Names (common patterns and formats)
  - Physical addresses (US addresses)
  - Social Security Numbers (SSN) - if detected
  - Credit card numbers - if detected

2.2 Detection Methods
---------------------
✓ Hybrid approach combining:
  - Regex patterns for structured data (emails, phone numbers)
  - Named Entity Recognition (NER) using spaCy or similar for names
  - LLM-based detection for context-aware identification (optional enhancement)
  - Address parsing using regex and validation

2.3 User Interface
------------------
✓ Web-based application (React/Next.js or Streamlit)
✓ File upload interface for PDFs
✓ Text input area for plain text
✓ Side-by-side or toggle view showing:
  - Original content
  - Redacted content
✓ Redaction statistics panel showing:
  - Count of each PII type detected
  - List of redacted items (optional)
✓ Download functionality for redacted output:
  - Redacted text file (.txt)
  - Redacted PDF file (.pdf)

2.4 Bonus Features (In Scope)
------------------------------
✓ Toggle controls for each PII type:
  - Enable/disable redaction for emails
  - Enable/disable redaction for phone numbers
  - Enable/disable redaction for names
  - Enable/disable redaction for addresses
✓ Customizable redaction labels:
  - Option to show as [EMAIL_1], [PHONE_1], etc.
  - Option to show as black boxes (████)
  - Option for user-defined custom labels
✓ Export functionality:
  - Download redacted text as .txt
  - Download redacted PDF (maintaining formatting where possible)

2.5 Documentation & Reasoning
-----------------------------
✓ Clear documentation panel explaining:
  - What PII types were detected
  - How many instances of each type
  - Reasoning/logic used for detection
✓ Logs or audit trail of redaction process

5. TECHNICAL APPROACH
================================================================================

5.1 Technology Stack
--------------------
• Frontend: React/Next.js OR Streamlit (for rapid prototyping)
• Backend: Python (Flask/FastAPI) if using React frontend
• PDF Processing: PyPDF2/pypdf or pdfplumber
• PII Detection:
  - Regex patterns (re module)
  - spaCy for NER (names, locations)
  - Optional: OpenAI API for enhanced detection
• Text Processing: Python string manipulation, regex

5.2 Architecture
-----------------
• Single-page application or simple multi-page app
• Client-side processing where possible
• Server-side processing for PDF handling
• Stateless design (no persistent storage)

5.3 Detection Strategy
----------------------
1. Preprocessing: Extract text from PDF or use input text
2. Multi-pass detection:
   - Pass 1: Regex for structured data (emails, phones)
   - Pass 2: NER for names and locations
   - Pass 3: Context-aware validation
3. Redaction: Replace detected PII with chosen format
4. Output generation: Create redacted text/PDF

================================================================================
6. ASSUMPTIONS & LIMITATIONS
================================================================================

Assumptions:
------------
• Input PDFs are text-based (not scanned images requiring OCR)
• English language content only
• Standard US address formats
• Users will provide valid PDF/text files

Known Limitations:
------------------
• May have false positives in PII detection
• Complex PDF layouts may not preserve perfectly
• Large PDF files (>10MB) may have performance issues
• Some edge cases in phone number formats may be missed
• Name detection may not catch all variations
